# MORVA 기능 명세서

## 1. 서비스 개요

MORVA는 **일일 계획 관리 서비스**로, 사용자가 매일의 할 일을 체계적으로 관리할 수 있도록 돕습니다.

- **기술 스택**: Spring Boot 3.5 + Java 21 + MySQL
- **아키텍처**: DDD (Domain-Driven Design), Bounded Context 분리
- **인증 방식**: JWT 토큰 기반 (Access + Refresh), OAuth2 (Google, Naver)

---

## 2. Bounded Context 구조

| BC | 설명 | 애그리거트 |
|----|------|-----------|
| **Account** | 인증 및 사용자 관리 | User, AuthCredential |
| **Planning** | 일일 계획 및 시간표 | DailyPlan, Task, Timetable, Slot |
| **Visioning** | 미래 비전 관리 | FutureVision |

---

## 3. Account BC — 인증/사용자 관리

### 3.1 로컬 회원가입

- 이메일, 계정 ID, 프로필 이름, 프로필 이미지, 비밀번호로 회원가입
- 계정 ID와 이메일 중복 검증
- 프로필 이미지는 S3에 업로드
- 비밀번호는 BCrypt로 암호화 저장

### 3.2 로컬 로그인

- 계정 ID + 비밀번호로 인증
- 성공 시 Access Token + Refresh Token 발급
- Refresh Token은 Redis에 저장

### 3.3 OAuth2 로그인 (Google, Naver)

- OAuth2 인증 후 사용자 정보 조회
- 기존 사용자면 토큰 발급, 신규 사용자면 프로필 설정 페이지로 리다이렉트
- 프로필 설정 완료 후 토큰 발급

### 3.4 토큰 재발급

- Refresh Token으로 새 Access Token + Refresh Token 발급
- 기존 Refresh Token은 폐기

## 4. Planning BC — 일일 계획

### 4.1 DailyPlan (일일 계획)

- 사용자별 날짜당 1개의 DailyPlan 생성 (유니크 제약)
- DailyPlan 생성 시 Timetable도 함께 생성 (도메인 이벤트)

### 4.2 MindSweep (마인드 스윕)

할 일을 자유롭게 나열하는 단계입니다.

| 기능 | 설명 |
|------|------|
| 추가 | DailyPlan에 일반(GENERAL) 태스크 추가 |
| 조회 | DailyPlan의 일반 태스크 전체 조회 |
| 수정 | 태스크 내용 변경 (완료된 태스크는 수정 불가) |
| 삭제 | 태스크 삭제 (완료된 태스크는 삭제 불가) |
| 완료 | 태스크를 완료 상태로 변경 |
| 미완료 | 완료된 태스크를 다시 미완료로 변경 |

### 4.3 Morning Task (아침 태스크)

아침 루틴을 관리하는 기능입니다.

| 기능 | 설명 |
|------|------|
| 추가 | 날짜 기반으로 MORNING 태스크 추가 (DailyPlan 없으면 자동 생성) |
| 조회 | 해당 날짜의 아침 태스크 전체 조회 |
| 수정 | 태스크 내용 변경 (완료된 태스크는 수정 불가) |
| 삭제 | 태스크 삭제 (완료된 태스크는 삭제 불가) |
| 완료 | 태스크를 완료 상태로 변경 |
| 미완료 | 완료된 태스크를 다시 미완료로 변경 |

**MindSweep과의 차이점**:
- MindSweep: `dailyPlanId` 기반으로 동작, TaskType = `GENERAL`
- Morning Task: `targetDate` 기반으로 동작, TaskType = `MORNING`, DailyPlan 자동 생성

### 4.4 Top Pick (핵심 태스크 선택)

MindSweep에서 나열한 태스크 중 **최대 3개**를 핵심 태스크로 선택합니다.

| 기능 | 설명 |
|------|------|
| 선택 | 태스크를 Top Pick으로 지정 (예상 시간 + 메모 입력) |
| 해제 | Top Pick 선택 해제 |
| 조회 | Top Pick으로 선택된 태스크 전체 조회 |

**비즈니스 규칙**:
- 최대 3개까지만 Top Pick 선택 가능
- Top Pick 선택 시 예상 소요 시간(분)과 메모 필수 입력
- Top Pick 선택/해제 시 Timetable에 도메인 이벤트 전파

### 4.5 Timetable (시간표)

Top Pick과 일반 태스크를 시간 슬롯에 배치합니다.

| 기능 | 설명 |
|------|------|
| 조회 | DailyPlan에 연결된 타임테이블 조회 |
| Top Pick 슬롯 할당 | Top Pick 태스크를 시간 슬롯에 배정 |
| 일반 태스크 슬롯 할당 | 모든 Top Pick 할당 후 일반 태스크 배정 가능 |
| 직접 슬롯 추가 | 태스크 없이 직접 내용과 시간을 지정하여 슬롯 추가 |
| 슬롯 시간 변경 | 기존 슬롯의 시작/종료 시간 변경 |
| 슬롯 제거 | 슬롯 삭제 |

**비즈니스 규칙**:
- 슬롯 간 시간 겹침 불가
- 시작 시간 < 종료 시간 검증
- 모든 Top Pick이 할당된 후에만 일반 태스크 할당 가능
- Top Pick 슬롯은 topPickTotal 수를 초과할 수 없음

## 5. Visioning BC — 미래 비전

### 5.1 FutureVision (미래 비전)

사용자의 장기 목표와 비전을 이미지와 텍스트로 관리합니다.

| 기능 | 설명 |
|------|------|
| 생성 | 주간 비전 이미지 + 연간 비전 이미지 + 설명으로 생성 |
| 전체 조회 | 사용자의 미래 비전 전체 정보 조회 |
| 주간 비전 조회 | 주간 비전 이미지만 조회 |
| 연간 비전 조회 | 연간 비전 이미지 + 설명 조회 |
| 주간 비전 수정 | 주간 비전 이미지 교체 |
| 연간 비전 수정 | 연간 비전 이미지 + 설명 수정 |

**비즈니스 규칙**:
- 사용자당 1개의 FutureVision만 생성 가능
- 이미지는 S3에 업로드, 수정 시 기존 이미지 삭제 후 새 이미지 업로드
- 연간 비전 설명은 최대 100자

## 6. 크로스 BC 통신 (도메인 이벤트)

| 이벤트 | 발행 | 수신 | 설명 |
|--------|------|------|------|
| `DailyPlanCreatedEvent` | DailyPlan | Timetable | DailyPlan 생성 시 Timetable 자동 생성 |
| `TaskTopPickedEvent` | DailyPlan | Timetable | Top Pick 선택 시 topPickTotal 증가 |
| `TaskTopPickUnpickedEvent` | DailyPlan | Timetable | Top Pick 해제 시 해당 슬롯 제거 + topPickTotal 감소 |
| `SlotRescheduledEvent` | Timetable | DailyPlan | 슬롯 시간 변경 시 관련 정보 동기화 |

모든 이벤트는 `@TransactionalEventListener(phase = BEFORE_COMMIT)`으로 처리됩니다.

## 7. 도메인 모델 관계도

```
[Account BC]
└── User (1:N) ──── AuthCredential

[Planning BC]
├── DailyPlan (1:N) ──── Task (1:1) ──── TopPickDetail (선택적)
│       │                  │
│       │                  ├── TaskType: GENERAL (마인드스윕)
│       │                  └── TaskType: MORNING (아침 태스크)
│       │
│       └── (DailyPlanId로 연결, 도메인 이벤트 동기화)
│
└── Timetable (1:N) ──── Slot

[Visioning BC]
└── FutureVision (사용자당 1개)
```

---

## 8. 인증/인가

| 구분 | 설명 |
|------|------|
| 인증 방식 | JWT (Access Token + Refresh Token) |
| Access Token | 요청 헤더 `Authorization: Bearer {token}` |
| Refresh Token | Redis 저장, 재발급 시 교체 |
| 공개 API | `/auth/signup`, `/auth/login`, `/auth/oauth/profile-setup`, `/auth/reissue` |
| 인증 필수 API | 그 외 모든 API |
| 소유권 검증 | 모든 리소스 접근 시 `userId` 기반 소유권 확인 |
